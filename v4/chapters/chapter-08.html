第 08 章 · 偏好优化

GRPO 与新型偏好优化方法

用组内比较降低价值模型依赖

GRPO：组相对策略优化

本章导读

本章把原材料中的相关内容重新组织为学习版讲解，重点追求清晰、连贯和可复习。原始细节可回到 V3 结构化文档或 arXiv PDF 查阅。

学习目标

  • 理解 GRPO 的组内相对思想
  • 知道它为何适合推理模型训练
  • 比较 PPO、DPO 和 GRPO 的取舍

核心概念

  • 智能体Agent

    能够感知状态、规划行动、调用工具并迭代完成任务的系统单元。

  • 标记 / tokenToken

    语言模型处理文本的基本离散单位。

  • TransformerTransformer

    以自注意力为核心的序列建模架构。

  • 注意力Attention

    根据查询、键和值计算上下文加权表示的机制。

  • 强化学习Reinforcement Learning

    通过奖励信号优化策略的学习范式。

  • 奖励模型Reward Model

    把候选输出映射为偏好或质量分数的模型。

  • 策略Policy

    在状态下选择动作或生成 token 的概率分布。

  • 轨迹Trajectory

    智能体与环境交互形成的状态、动作、观察和奖励序列。

组相对的想法

组相对策略优化（Group Relative Policy Optimization / GRPO）会为同一 prompt 采样一组回答，再用组内相对表现估计优势。这样可以减少对单独价值模型的依赖。

为什么适合推理

推理任务常常可以通过规则、验证器或最终答案判断正确性。同一题目的多条思路可以相互比较，形成更稳定的训练信号。

工程取舍

GRPO 仍然需要采样多个候选，因此推理成本不低。它的关键在于奖励设计、组大小、采样温度和结果验证。

关键图表 / 表格：三种偏好优化直觉

方法核心数据适合场景
PPO在线 rollout + 奖励需要探索和复杂反馈

GRPO同题多候选组内比较推理、数学、代码等可验证任务

公式解释

GRPO 的重点不是单个回答的绝对分数，而是同组回答相对均值的优势。V4 用这个解释替代易损坏的公式抄录。

小结

  • GRPO 通过组内比较获得训练信号。
  • 它适合可验证推理，但采样和奖励设计仍然关键。

思考题

  1. 为什么同一 prompt 下多采样可以帮助训练？
  2. 组大小过小或过大会分别带来什么问题？
