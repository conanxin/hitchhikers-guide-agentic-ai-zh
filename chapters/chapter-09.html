Online DPO

原 PDF p.174

偏好优化变体

本章涵盖了使用不同目标、数据扩展或替换 DPO 的一系列方法 假设或架构权衡。每个解决了标准离线 DPO 的特定限制： 分布偏移（在线 DPO）、配对数据的需求（KTO）、对噪声标签的过度拟合（IPO）、 参考模型内存成本 (ORPO) 或训练复杂度（Best-of-N）。

在线数据保护官

动机

标准 DPO 的主要限制：偏好数据是由不同的模型生成的（通常 较旧的检查点甚至不同的型号系列）。随着训练的进行，策略会生成 文本看起来与训练对完全不同→损失正在不相关的分布上进行优化。 在线 DPO 解决方案 [195]：每次从当前策略生成新的偏好对 步骤，用奖励模型来判断它们，然后应用 DPO 损失。

算法

  1. 根据当前 πθ 根据提示生成 K 个响应

  2. 使用奖励模型 rphi 对所有响应进行评分

  3. 创建配对：最高得分 = 选择，最低得分 = 拒绝

  4. 对这些新对应用 DPO 损失

5.重复（每一步产生新的一代）

在线 DPO = 两全其美

• 来自DPO：简单的监督损失，无价值函数，无GAE，稳定优化

• 来自 PPO：符合政策的数据、数据集之外的自我改进、无分布变化

• 与 GRPO 的主要区别：使用 DPO 损失（基于对）而不是 PPO 损失（每个样本） 优势）

权衡：需要奖励模型（DPO 不需要），但没有价值头（PPO 需要）。中间立场 复杂性。

KTO：卡尼曼-特沃斯基优化

动机

DPO 需要成对的偏好：对于同一个提示，您需要好的响应和坏的响应。在 在实践中，大多数反馈都是不成对的：用户对个别响应给出赞成/反对的意见，但没有 匹配的一对。 KTO 的见解[11]：使用前景理论（来自行为经济学）。人类更容易感受到损失 强于收益。 “拇指向下”应该产生比“拇指向上”更强的梯度。

通过前景理论获得 KTO 直觉

期望的响应 (yw)：模型通过增加概率来获得“效用”。但与 收益递减——一旦这种情况已经很可能发生，就不要再努力了。 不良反应 (yl)：损失厌恶意味着对生成不良文本的惩罚进行加权 比对好文字的奖励更强烈。默认值：λl = 1.0，λw = 1.0，但您可以设置 λl > λw。 主要优点：每个训练示例都是独立的！无需寻找匹配对。可以使用 直接赞成/反对数据。

KTO Data Format
Unlike DPO which needs: {"prompt": ..., "chosen": ..., "rejected": ...}
KTO only needs: {"prompt": ..., "completion": ..., "label": true/false}
This means you can use:

• 生产流量的好评/差评

• 来自论坛的赞成票/反对票

• 人类评分二值化（4-5 星 = 好，1-2 = 差）

• 任何每次响应的质量信号

IPO - Identity Preference Optimization

• 大规模生产赞成/反对数据

• 一类占主导地位（例如，90% 好，10% 坏）——KTO 更好地处理不平衡

IPO——身份偏好优化

动机

DPO有一个简并的解决方案：它可以通过在选择和选择之间做余量来实现零损失 拒绝无限大。实际上，这意味着 DPO 过度拟合——将所选概率推至 1 并且 拒绝为0，记忆训练数据。 IPO 的解决方案 [12]：使用针对特定目标的平方损失，而不是 log-sigmoid（饱和） 保证金。损耗在有限间隙（而不是无穷大）处最小化。

πref(yl|x) −1

IPO 与 DPO：通过目标利润率进行正规化

DPO： σ(margin) → 1 最佳。裕度 → 无穷大。没有自然的停止点。 IPO：保证金 → 2β最佳。平方损失会惩罚过小和过大的利润。 结果：IPO 对嘈杂的标签更加稳健（标签错误的一对受到的影响有限），并且 概括性更好，因为它不需要记忆。

ORPO：优势比偏好优化

动机

到目前为止，所有方法都需要一个参考模型——要么作为单独的副本（双倍内存），要么隐式地 通过洛拉。 ORPO [13] 通过结合 SFT 和偏好对齐完全消除了引用 在单次损失中。 关键见解：使用生成选择与拒绝的优势比作为偏好信号。的 SFT 组件自然可以防止崩溃（不需要 KL 正则化）。

ORPO：SFT + 一次对齐

SFT 术语：训练模型以很好地生成所选响应（标准语言建模）。 优势比项：另外推动模型更倾向于选择而不是拒绝。优势比 是一种自然的对比，不需要参考模型。 为什么不需要参考？：SFT 损失已经将模型锚定到合理的文本。它 在其他方法中与 KL-to-reference 的作用相同。一种模型，一次前向传递，一次损失。 内存减少 50%！

Best-of-N Sampling (Rejection Sampling)

原 PDF p.179

• 从基础模型开始（尚未进行 SFT）— ORPO 同时进行 SFT

• 想要最简单的流水线：一个模型、一次损失、一次训练运行

• 从一开始就可获得良好的偏好数据

ORPO 限制

• 比 DPO/PPO 研究较少 — 70B+ 规模的经过验证的配方较少

• SFT 组件意味着它需要高质量的选择响应（而不仅仅是相对偏好） 恩斯）

• 更难调试：两个损耗分量可能会发生冲突

另请参阅：SimPO SimPO [183] 是另一种无参考偏好方法，它使用长度归一化对数 概率作为隐性奖励，完全消除参考模型。 它被覆盖在 由于其共享的无引用理念，第 6.9.8 节与其他 DPO 扩展并列。

N 次最佳采样（拒绝采样）

动机

有时最简单的方法会获胜。 Best-of-N [196] 在 RL 期间根本不需要任何训练 阶段 - 只需生成多个候选者并选择最好的一个。

算法

  1. 对于每个提示，根据策略生成 N 个响应（通常 N = 4–64）

  2. 使用奖励模型对所有回复进行评分

  3. 选择得分最高的答案

4.（可选）使用选定的响应作为下一次迭代的 SFT 数据

Best-of-N response :
y∗= arg
max
yi∼πθ(·|x) rϕ(x, yi)
(8.4)

为什么 Best-of-N 是一种合法的“强化学习”方法

在推理时：Best-of-N 可以在不改变模型权重的情况下提高输出质量。与 N = 64，胜率比贪婪提高 10-20%——有时匹配或超过 PPO。 作为训练方法（拒绝采样微调/RFT）：

  1. 生成许多响应，选择最佳响应

  2. 对所选响应进行 SFT

  3. 重复（迭代细化）

这就是训练多个生产模型的方式：比 PPO 更简单，几乎同样有效，完全
stable.

  1. 你们有可验证的奖励吗？ （数学/代码）→GRPO
