第 05 章 · 对齐基础

从 SFT 到 RLHF

让模型先会答，再学会答得更好

语言模型的强化学习基础SFT 最佳实践与技术

本章导读

本章把原材料中的相关内容重新组织为学习版讲解，重点追求清晰、连贯和可复习。原始细节可回到 V3 结构化文档或 arXiv PDF 查阅。

学习目标

  • 理解 SFT 的作用
  • 理解 RLHF 的基本流水线
  • 知道 SFT 数据质量如何影响后续 RL

核心概念

  • 智能体Agent

    能够感知状态、规划行动、调用工具并迭代完成任务的系统单元。

  • 标记 / tokenToken

    语言模型处理文本的基本离散单位。

  • 强化学习Reinforcement Learning

    通过奖励信号优化策略的学习范式。

  • 监督微调（SFT）Supervised Fine-Tuning

    用示范数据让模型学习期望响应格式和行为。

  • 奖励模型Reward Model

    把候选输出映射为偏好或质量分数的模型。

  • 策略Policy

    在状态下选择动作或生成 token 的概率分布。

  • 推理Inference

    模型在部署或测试时生成输出的过程。

  • 近端策略优化（PPO）PPO

    通过裁剪目标限制策略更新幅度的强化学习算法。

SFT 解决起点问题

监督微调（Supervised Fine-Tuning / SFT）用高质量示范回答教模型遵循指令、组织格式和覆盖常见任务。它不是终点，但会决定后续偏好优化的天花板。

RLHF 解决偏好问题

人类反馈强化学习（RLHF）通常包括采样候选、收集偏好、训练奖励模型、再用强化学习优化策略。它把“哪种回答更好”变成可训练信号。

数据格式很关键

聊天模板、completion-only mask、序列打包和多任务混合都会影响训练稳定性。V4 把这些工程细节放在学习路径中，而不是散落在 PDF 长页里。

公式解释

偏好优化的关键不是背公式，而是理解：模型会被鼓励提高胜出回答的概率，同时控制偏离参考模型的幅度。

小结

  • SFT 提供行为起点，RLHF 进一步塑造偏好。
  • 数据质量、模板和 mask 会显著影响训练结果。
